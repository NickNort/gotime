# Add non-finder module connection algorithms (horizontal, vertical, both)

### Overview

Introduce a new option to control how non-finder modules are grouped when rendering: horizontal (existing behavior), vertical, or both. Ensure runs only span contiguous dark modules outside finder regions so light modules are never filled. Expose this via the internal library, CLI flag, and HTTP API.

### Files to update

- `internal/qr/qr.go`
- `cmd/climain.go`
- `main.go`

### Library changes (`internal/qr/qr.go`)

1) Extend `Options` and defaults/validation

- Add `ModuleConnect string // "horizontal" | "vertical" | "both"`.
- Default to `"horizontal"`.
- Validate allowed values; error otherwise.
- Continue enforcing brand colors `#552048` and `#f8f2ec` as defaults.

2) Update `renderQR` to support connect modes for square/rounded

- Keep current per-cell drawing for `circle` and `diamond` shapes.
- For `square`/`rounded`:
  - horizontal: keep current row run grouping (unchanged).
  - vertical: add column run grouping (draw one rect per contiguous dark vertical run outside corners).
  - both: execute both passes (horizontal then vertical) to visually connect in both axes; each pass still only draws contiguous dark runs, so light modules remain unfilled.
- Continue skipping finder areas via `isInCorner`.

### CLI changes (`cmd/climain.go`)

- Add a new flag: `-connect` (and support both `-connect=value` and `-connect value`). Allowed: `horizontal`, `vertical`, `both`.
- Include in manual post-positional parsing like the existing flags.
- Pass through into `qr.Options{ ModuleConnect: *connect }`.

### HTTP API changes (`main.go`)

- Extend `QRRequest` with `ModuleConnect string \`json:"module_connect,omitempty"\``.
- Forward to `qr.GenerateSVG` via `qr.Options{ ModuleConnect: req.ModuleConnect, ... }`.

### Behavioral notes

- Horizontal/vertical/both only affect `square` and `rounded` `module_shape`; `circle` and `diamond` remain per-cell.
- Both-mode draws two passes; rectangles may overlap but never span light modules.
- Finder patterns remain excluded from grouping.

### Usage examples

- CLI: `gotime -module-shape=square -connect=vertical "hello"`
- HTTP: `{ "content": "hello", "module_shape": "rounded", "module_connect": "both" }`
